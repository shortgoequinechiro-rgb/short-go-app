BLOCK LAUNCH — Fix Immediately

Critical
1. Rotate all exposed API keys and secrets
.env.local is committed to Git with live keys: OpenAI, Resend, Supabase service role, Twilio SID + auth token, Stripe secret key, and test login credentials. Rotate every key immediately, then purge from Git history with git-filter-repo.
.env.local (lines 1-27)
Critical
2. Remove .env.local from Git and scrub history
Run: git rm --cached .env.local, verify .gitignore has it, then use git-filter-repo or BFG to purge all secrets from commit history. Anyone who cloned the repo has your keys.
.env.local, .gitignore
Critical
3. Configure real Stripe webhook secret
Currently set to "whsec_REPLACE_ME" — webhook signature verification will always fail. Get the real signing secret from Stripe Dashboard > Webhooks. Without this, anyone can forge billing events (fake subscriptions, bypass payment).
.env.local line 16, app/api/billing/webhook/route.ts:36
Critical
4. Fix XSS in contact form email template
User input (name, email, message, phone, practice name) is interpolated directly into HTML without escaping. Attacker can inject script tags via the contact form. Add an escapeHtml() utility and apply to all user-provided fields in email HTML.
app/api/contact/route.ts (lines 34, 38, 46, 51)
Critical
5. Fix overly permissive RLS on consent_forms, intake_forms, vet_authorizations
These tables have USING (true) policies — any unauthenticated user can read ALL consent/intake/vet auth records from ALL practitioners. Drop the blanket public SELECT policies and restrict to UUID-based access (owner_id in URL as token).
migrations/001_add_multi_tenancy.sql (lines 84-108), migrations/034_add_vet_authorizations.sql (lines 61-73)
Critical
6. BillingGate fails open on network error
If the ensure-practitioner API call fails (network issue, 500 error), the component sets ready=true and grants access to paid features. Users can access premium features during outages without paying. Add fallback to deny access on error.
app/components/BillingGate.tsx (lines 76, 111-114)
Critical
7. Add Terms of Service and Privacy Policy pages
No legal pages exist. You're collecting PII, medical records, payment info, and sending SMS. You need ToS and Privacy Policy before launch. Consider consulting a lawyer familiar with TCPA (SMS) and animal medical records.

HIGH — Must Fix Before Launch

Security

High
8. Add rate limiting to all public and sensitive endpoints
Only /api/contact (5/hr) and /api/owners (20/hr) are rate limited. Auth endpoints, SMS incoming webhook, public form endpoints, and AI generation have zero rate limiting. Add per-IP and per-user limits. Current in-memory store also resets on deploy.
middleware.ts (lines 10-12)
High
9. Add global error boundary (error.tsx)
No error boundary exists anywhere in the app. One unhandled error in any component crashes the entire app with no fallback UI. Create app/error.tsx with a user-friendly error page and recovery option.
app/ (missing error.tsx)
High
10. Add pagination to dashboard queries
loadOwners() and loadHorses() fetch ALL records with no limit. With 500+ owners/horses, this will cause memory issues and slow load times. Add cursor-based or offset pagination with lazy loading.
app/dashboard/page.tsx (lines 493-680)
High
11. Fix Stripe webhook idempotency and missing events
No deduplication by event ID — same webhook could be processed twice. Missing handlers for: payment_intent.failed, customer.subscription.paused. Returns 200 even if DB update fails.
app/api/billing/webhook/route.ts (lines 43-162)
High
12. Add grace period enforcement cron job
Grace period is SET when subscription cancels but never enforced. No cron job checks if grace period has expired. Users stay logged in with full access after grace period ends. Add a cron that checks and locks out expired accounts.
app/api/billing/webhook/route.ts (lines 107-130), BillingGate.tsx
High
13. Fix consent/intake e-signature replay vulnerability
Signature data is stored as plain base64 PNG — can be copied between forms. No timestamp binding, no form-hash binding. Add a signature hash that includes form ID + timestamp to prevent replay.
app/api/consent/submit/route.ts (line 55)
High
14. Fix SMS incoming webhook — no rate limiting or flood protection
An attacker sending 1000 SMS from the same number triggers 1000 API calls, each creating a notification and hitting Supabase. Add rate limiting per phone number and implement Twilio request signature validation.
app/api/sms/incoming/route.ts (lines 22-150)
High
15. Add error tracking (Sentry or similar)
No error monitoring in production. Unhandled errors, API failures, and crashes go unnoticed. Set up Sentry with source maps for both client and server errors.
High
16. OpenAI prompt injection risk in SOAP generation
User-provided quick notes are embedded directly in the AI prompt. A practitioner (or attacker) can inject instructions to manipulate clinical output. Use structured prompts with delimiters and add output validation.
app/api/generate-soap/route.ts (lines 45-70)
High
17. Offline sync has silent data loss on conflict
syncPendingData() inserts without conflict checking. If unique violation (23505), the record is deleted from local DB silently — user never knows their data was lost. Show a user-facing conflict resolution UI.
app/lib/offlineDb.ts (lines 279-396)
High
18. Remove NEXT_PUBLIC_BYPASS_BILLING from production
Test scripts set this flag to bypass billing checks. If accidentally deployed to production, all subscription enforcement is disabled. Add a guard that only checks this flag in development.
package.json (lines 10-11), BillingGate.tsx
High
19. Notifications table RLS allows any user full access
Policy uses USING (true) for ALL operations. Combined with exposed service key, any user can read/modify all notifications. Restrict to practitioner_id = auth.uid().
migrations/029_add_notifications.sql (line 18)
High
20. Communication log allows unrestricted insert
FOR INSERT WITH CHECK (true) means anyone with the service key can insert fake communication logs for any practitioner. Add practitioner_id validation.
migrations/028_add_communication_log.sql (line 24)
High
21. Create .env.example with all required variables documented
No documentation of required environment variables. New developers or deployment environments have no reference. Create .env.example with placeholder values and descriptions.

MEDIUM — Should Fix Before or Shortly After Launch

Security & Data

Medium
22. Remove unsafe-eval and unsafe-inline from CSP headers
Current CSP allows inline scripts and eval(), which weakens XSS protection. Use nonces for inline scripts/styles instead.
middleware.ts (lines 59-60)
Medium
23. Add CSRF protection on state-changing endpoints
No CSRF tokens on POST/PUT/DELETE routes. Cross-origin requests could trigger unintended invoice creation, SMS sending, or appointment changes.
Medium
24. Validate file extensions against MIME type in uploads
Document upload validates MIME type but not extension. A file named "document.pdf.exe" with a spoofed MIME type could bypass checks. Cross-check extension and MIME, plus validate magic bytes.
app/api/owners/[ownerId]/documents/route.ts (lines 104-118)
Medium
25. Use a phone number parsing library for SMS normalization
Current phone matching uses fragile string manipulation with multiple format guesses. Non-US numbers will break. Use libphonenumber-js for proper E.164 normalization.
app/api/sms/incoming/route.ts (lines 54-79)

Functionality

Medium
26. Fix vet auth expiration off-by-one
Uses .lt('expires_at', today) so authorizations remain "active" on their expiration date. Should use .lte() to expire on the date itself.
app/api/vet-auth/route.ts (line 26)
Medium
27. Add transaction safety to multi-animal intake form submission
Submitting multiple animals has no database transaction. If animal 2 fails, animal 1 is already created with no rollback. Use Supabase's rpc() for transactional inserts.
app/api/intake/submit/route.ts (lines 67-143)
Medium
28. Add retry logic to external API calls (Stripe, OpenAI, Resend, Twilio)
All external API calls have no retry on failure. A single timeout or 429 rate limit silently fails the operation. Add exponential backoff with 2-3 retries.
Medium
29. Fix reminder cron timezone edge cases
Uses UTC dates for "tomorrow" calculation. Practitioners in UTC-8 could have reminders sent at wrong times or not at all. Store and use practitioner timezone.
app/api/reminders/send/route.ts (lines 169-192)
Medium
30. Add loading states to dashboard and horse detail tabs
Multiple async fetches (owners, horses, visits) have no loading indicators. Users see blank UI while data loads. Add skeleton loaders or spinners.
app/dashboard/page.tsx, app/horses/[id]/page.tsx
